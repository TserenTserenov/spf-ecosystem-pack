# Онтология: Экосистема развития интеллекта

> Онтология домена по SPF.SPEC.002.
> Полный реестр типов сущностей, ключевых понятий и связей для Pack ECO.
> Каждое понятие привязано к родительскому понятию из базовой онтологии SPF (SPF.SPEC.002, секция 2).

---

## 1. Виды сущностей

> Все типы, используемые в этом Pack: базовые (из SPF) + расширенные (определённые этим Pack).
> **Обязательная колонка «FPF-понятие»** — родительское понятие из базовой онтологии SPF.

### 1.1. Базовые виды (из SPF)

| Код | Вид | FPF-понятие | Определение | ≠ (что это НЕ) |
|-----|-----|-------------|-------------|-----------------|
| `D` | Различение | A.7 Strict Distinction | Концептуальная граница, нарушение которой создаёт неустранимую путаницу | ≠ факт, ≠ определение |

> Виды `M` (Method), `WP` (Work Product), `FM` (Failure Mode), `MAP` (Map), `CHR` (Characteristic), `SOTA` (SoTA) определены в SPF, но пока не используются в этом Pack.

### 1.2. Расширенные виды (планируемые)

> Pack находится на ранней стадии. По мере наполнения здесь появятся расширенные виды.

## 2. Глоссарий домена

> Ключевые понятия предметной области. Для полных определений — см. [01A-bounded-context.md](01A-bounded-context.md).
> **Обязательная колонка «Родительское понятие (SPF)»** — к какому универсальному понятию из базовой онтологии SPF относится этот доменный термин.

| Термин | Определение | Родительское понятие (SPF) | Связан с |
|--------|-------------|---------------------------|----------|
| **Экосистема** | Связанная совокупность систем (платформа, среды, сервисы) и сообществ, обеспечивающих бесшовный путь развития интеллекта. Экосистема живёт по правилам, знаниям и методологиям (нормативный контекст), но методология — не компонент экосистемы, а её контекст | U.System | D.ECO.001 |
| **Компонент экосистемы** | Отдельный ресурс/сервис/сообщество, выполняющий определённую функцию в контексте целого | U.Holon | D.ECO.002 |
| **Бесшовность** | Отсутствие разрывов и барьеров между этапами и компонентами экосистемы; качество переходов | U.Characteristic | D.ECO.003 |
| **Путь развития** | Адаптивная траектория прохождения через компоненты экосистемы; определяется состоянием созидателя | U.Flow | D.ECO.004 |
| **Точка входа** | Место первого контакта с экосистемой; может быть событием, сообществом, любым компонентом | U.Boundary + U.Interaction | D.ECO.005 |
| **Переход** | Смена одного компонента на другой в рамках пути; ключевой объект для измерения бесшовности | U.Interaction | — |
| **Качество экосистемы** | Связность целого, конверсия переходов; не сводится к качеству отдельных компонентов | U.Characteristic | D.ECO.006 |
| **Культура экосистемы** | Совокупность методов и практик, принятых внутри экосистемы. Не компонент (не система), а нормативный контекст, определяющий как участники действуют и взаимодействуют | U.Episteme | D.ECO.007 |

## 3. Отношения между видами

| Субъект | Отношение | Объект | Пример |
|---------|-----------|--------|--------|
| Экосистема | содержит → | Компонент | Экосистема → курс, сообщество, сервис |
| Компонент | связан через → | Переход | Компонент A → переход → Компонент B |
| Путь развития | проходит через → | Компонент | Путь → точка входа → компонент 1 → компонент 2 |
| Бесшовность | измеряет ← | Переход | Бесшовность ← качество конкретного перехода |
| Точка входа | является → | Компонент | Вебинар = точка входа и компонент |

## 4. Иерархия типов

```
Экосистема (U.System)
├── Компоненты (U.Holon)
│   ├── Платформа (общая система, shared)
│   ├── Среда / IWE (личная система, personal)
│   ├── Ресурс (курс, книга, база знаний)
│   ├── Сервис (инструмент)
│   └── Сообщество (клуб, группа, форум)
├── Переходы (U.Interaction)
│   ├── Автоматический (по завершении)
│   └── Навигируемый (по состоянию)
└── Точки входа (U.Boundary)
    ├── Событийная (вебинар, мероприятие)
    ├── Контентная (статья, пост)
    └── Социальная (рекомендация, сообщество)
```
